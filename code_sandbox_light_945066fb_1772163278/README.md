# 🏥 介護記録作成アシスタント - Google Apps Script版

完全にGoogle Drive上で動作する介護記録作成支援ツールです。音声入力からAIで介護記録を自動生成し、Google Sheetsに保存します。

## ✨ 主な機能

### 🎤 音声入力
- リアルタイム音声認識（Web Speech API）
- iPad/iPhone/Safari完全対応
- 連続録音・自動追記
- 介護用語の自動変換

### 🤖 AI記録文生成
- Google Gemini API統合
- 3種類の記録形式
  - 📝 **経過観察記録** - 日常の介護記録
  - 📊 **モニタリング記録** - 計画評価用
  - 📋 **支援計画書用** - ケアプラン作成用
- 客観的・簡潔な文章を自動生成

### 📚 履歴管理
- Google Sheetsに自動保存
- 全文検索機能
- 日時・タイプ別フィルタ
- クリック一つで過去の記録を呼び出し

### 📖 介護用語辞書
- デフォルト70語収録
- カスタム用語の追加・編集・削除
- 音声認識の精度向上
- Google Sheetsで管理

### 📱 完全レスポンシブ
- iPad/iPhone最適化
- タッチ操作対応
- どこからでもアクセス可能
- ブックマークでアプリのように使用

## 🚀 セットアップ手順

### **STEP 1: プロジェクト作成**

1. [Google Apps Script](https://script.google.com/)を開く
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「介護記録作成アシスタント」に変更

### **STEP 2: コードの配置**

#### **Code.gs（バックエンド）**
1. 左側のファイル一覧で「コード.gs」をクリック
2. このリポジトリの `Code.gs` の内容を全てコピー
3. エディタに貼り付けて保存（💾アイコン）

#### **index.html（フロントエンド）**
1. 左側のファイル一覧の「+」ボタン → 「HTML」を選択
2. ファイル名を「index」にする
3. このリポジトリの `index.html` の内容を全てコピー
4. エディタに貼り付けて保存

### **STEP 3: Google Gemini API Key取得**

1. [Google AI Studio](https://aistudio.google.com/app/apikey)にアクセス
2. Googleアカウントでログイン
3. 「**Create API Key**」ボタンをクリック
4. 「Create API key in new project」を選択
5. 表示されたAPIキー（`AIza...`で始まる）をコピー
   - ⚠️ **このキーは再表示できないので、必ず保存してください**

### **STEP 4: ウェブアプリとしてデプロイ**

1. 右上の「デプロイ」→「新しいデプロイ」をクリック
2. 種類の選択で「ウェブアプリ」を選択
3. 設定:
   - **説明**: 介護記録作成アシスタント v1.0
   - **次のユーザーとして実行**: 自分
   - **アクセスできるユーザー**: 自分のみ（または必要に応じて変更）
4. 「デプロイ」をクリック
5. 初回は承認が必要:
   - 「アクセスを承認」をクリック
   - アカウントを選択
   - 「詳細」→「（プロジェクト名）に移動」をクリック
   - 「許可」をクリック
6. 表示された**ウェブアプリのURL**をコピー
   - 例: `https://script.google.com/macros/s/XXXXX/exec`

### **STEP 5: 初回設定**

1. コピーしたURLをブラウザ（Safari推奨）で開く
2. 黄色い「API設定」カードが表示される
3. STEP 3でコピーしたGemini API Keyを貼り付け
4. 「💾 保存してテスト」ボタンをクリック
5. 「✅ 接続成功！」と表示されたら完了

### **STEP 6: ブックマーク登録**

#### **iPad/iPhoneの場合:**
1. Safariで共有ボタン（□↑）をタップ
2. 「ホーム画面に追加」をタップ
3. 名前を「介護記録」などに変更
4. 追加ボタンをタップ
5. ホーム画面にアイコンが追加される

#### **PCの場合:**
1. ブラウザのブックマークに追加
2. 次回からブックマークで即アクセス

## 📖 使い方

### **基本ワークフロー**

```
1. 🎤 マイクボタンをタップ → 話す
   ↓
2. ⏹️ 再度タップで停止
   ↓
3. ✍️ 記録タイプを選択（経過観察/モニタリング/計画書）
   ↓
4. ⏳ AI処理（30秒）
   ↓
5. 📋 生成された記録文をコピー
   ↓
6. 💾 保存（履歴に追加）
```

### **音声入力のコツ**

- 📍 **静かな環境**で使用
- 🎙️ **マイクに近づく**（30cm以内推奨）
- 🗣️ **ゆっくり明瞭**に話す
- ✋ **句読点は自動**挿入されるので不要
- 📝 **専門用語**は辞書に登録すると精度向上

### **用語辞書の活用**

#### **カスタム用語の登録例:**

| 誤認識語 | 正しい表記 | 用途 |
|---------|----------|------|
| たなかさん | 田中様 | 利用者名 |
| にかい | 2階フロア | 施設内場所 |
| ぴーてぃー | PT（理学療法） | 専門用語 |

#### **登録手順:**
1. 「📖 用語辞書管理」カードを開く
2. 誤認識語と正しい表記を入力
3. 「➕ 追加」ボタンをクリック
4. 即座に反映される

### **履歴から記録を再利用**

1. 左サイドバーの記録をクリック
2. 内容が自動読み込み
3. 編集して再保存 or そのままコピー

### **検索機能**

```
検索ボックスに「利用者名」「日付」「キーワード」を入力
→ 該当する記録のみ表示
```

## 📊 データ構造

### **Google Sheetsの構成**

プロジェクトを実行すると、自動的に2つのシートが作成されます:

#### **1. 介護記録履歴**
| カラム | 説明 |
|--------|------|
| ID | ユニークID（UUID） |
| タイトル | 記録のタイトル |
| 作成日時 | 最初の作成日時 |
| 更新日時 | 最終更新日時 |
| 記録タイプ | standard/monitoring/plan |
| 入力テキスト | 元の文字起こし |
| 整理済テキスト | AI整理後のテキスト |
| 最終記録文 | 生成された記録文 |
| 職員名 | 作成者名（オプション） |
| 利用者名 | 対象利用者名（オプション） |

#### **2. 用語辞書**
| カラム | 説明 |
|--------|------|
| 誤認識語 | 音声認識される誤った表現 |
| 正しい表記 | 変換後の正しい表現 |
| カテゴリ | ADL、バイタル等の分類 |
| 登録日時 | 登録日時 |

## 🔐 セキュリティ

### **API Key管理**
- Google Apps Scriptのプロパティサービスに暗号化保存
- 外部には公開されない
- 作成者のみアクセス可能

### **データ保存**
- 全てのデータは作成者のGoogle Driveに保存
- 外部サーバーへの送信なし
- Google AI に一時送信（処理のみ）
- 処理後は自動破棄

### **アクセス制御**
- デフォルトは作成者のみアクセス可能
- 必要に応じて共有設定を変更可能

## ⚠️ 注意事項

### **音声認識**
- Safariのみ対応（Chrome on iOSは非対応）
- 長時間録音は自動停止する場合あり
- バックグラウンドでは動作停止
- プライベートブラウズでは制限あり

### **データ管理**
- 定期的にGoogle Sheetsをバックアップ推奨
- 重要な記録は必ず内容確認後に使用
- AI生成文は必ず人が最終確認すること

### **個人情報**
- 利用者の個人情報を含む記録の取扱いに注意
- 共有PCでは使用後にログアウト推奨
- API Keyは第三者に漏らさない

## 🐛 トラブルシューティング

### **Q1: 音声入力が動かない**

**A**: 以下を確認してください
1. Safariを使用しているか
2. 設定 > Safari > マイク: 許可
3. ページをリロード
4. マイクボタンを再度タップ

### **Q2: API接続エラー**

**A**: 
1. API Keyが正しいか確認（`AIza`で始まる）
2. [Google Cloud Console](https://console.cloud.google.com/)で「Generative Language API」が有効か確認
3. インターネット接続を確認

### **Q3: 記録が保存されない**

**A**:
1. Google Apps Scriptの権限を確認
2. Googleアカウントでログインしているか確認
3. ブラウザのキャッシュをクリア

### **Q4: 履歴が表示されない**

**A**:
1. Google Sheetsが作成されているか確認
2. 「介護記録履歴」シートが存在するか確認
3. ページをリロード

### **Q5: デプロイURLにアクセスできない**

**A**:
1. デプロイ時に承認を完了したか確認
2. デプロイの「アクセスできるユーザー」設定を確認
3. 新しいデプロイを作成してみる
