## 事前準備

### **必要なもの:**

- ✅ Googleアカウント（無料）
- ✅ インターネット接続
- ✅ PC/Mac（初回セットアップ用）
- ✅ iPad/iPhone（実際の使用用、オプション）

### **推奨環境:**

- **ブラウザ**: Chrome、Safari、Edge
- **音声入力**: Safari（iPad/iPhone）
- **所要時間**: 約15分

## STEP 1: Google Apps Scriptプロジェクト作成

### **1-1. Google Apps Scriptにアクセス**

1. ブラウザで [https://script.google.com/](https://script.google.com/) を開く
2. Googleアカウントでログイン（未ログインの場合）

### **1-2. 新しいプロジェクトを作成**

1. 左上の「**新しいプロジェクト**」ボタンをクリック
2. 新しいエディタ画面が開く
3. 左上の「無題のプロジェクト」をクリック
4. プロジェクト名を「**介護記録作成アシスタント**」に変更
5. 「名前を変更」をクリック

### **1-3. デフォルトのコードを削除**

エディタに表示されている以下のコードを全て削除:

```javascript
function myFunction() {

}
```

---

## STEP 2: コードの配置

### **2-1. Code.gs（バックエンド）を配置**

#### **手順:**

1. このリポジトリの `Code.gs` ファイルを開く
2. 全ての内容をコピー（Ctrl+A → Ctrl+C / Cmd+A → Cmd+C）
3. Google Apps Scriptのエディタに貼り付け（Ctrl+V / Cmd+V）
4. 上部の💾アイコン（または Ctrl+S / Cmd+S）で保存

#### **確認ポイント:**

✅ ファイル名が「**Code.gs**」になっている  
✅ 行数が約400行前後  
✅ エラー表示がない（赤いアンダーラインがない）

---

### **2-2. index.html（フロントエンド）を配置**

#### **手順:**

1. 左側のファイル一覧の「**+**」ボタンをクリック
2. 「**HTML**」を選択
3. ファイル名入力欄に「**index**」と入力
4. 「OK」をクリック
5. このリポジトリの `index.html` ファイルを開く
6. 全ての内容をコピー
7. Google Apps Scriptの `index.html` エディタに貼り付け
8. 保存（💾アイコン）

#### **確認ポイント:**

✅ ファイル名が「**index.html**」になっている  
✅ 行数が約800行前後  
✅ `<!DOCTYPE html>` で始まっている  
✅ `</html>` で終わっている

---

### **2-3. ファイル構成の確認**

左側のファイル一覧に以下の2つのファイルがあることを確認:

```
📁 介護記録作成アシスタント
  ├─ 📄 Code.gs
  └─ 📄 index.html
```

## STEP 3: Google Gemini APIキー取得

### **3-1. Google AI Studioにアクセス**

1. ブラウザで [https://aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey) を開く
2. Googleアカウントでログイン（同じアカウント推奨）

### **3-2. APIキーを作成**

1. 「**Get API Key**」または「**Create API Key**」ボタンをクリック
2. ダイアログが表示される:
   - 既存のプロジェクトを選択 または
   - 「**Create API key in new project**」を選択（推奨）
3. 「Create」ボタンをクリック

### **3-3. APIキーをコピー**

1. 表示されたAPIキー（`AIzaSy...`で始まる長い文字列）が表示される
2. 「**Copy**」ボタンをクリック
3. **⚠️ 重要:** このキーは再表示できないので、安全な場所に保存しておく
   - 例: パスワード管理アプリ、メモ帳など

#### **APIキーの形式例:**

```
AIzaSyABCDEFGHIJKLMNOPQRSTUVWXYZ1234567890
```

#### **確認ポイント:**

✅ `AIza` で始まっている  
✅ 39文字（約）の長さ  
✅ コピー済み

---

## STEP 4: ウェブアプリとしてデプロイ

### **4-1. デプロイ設定を開く**

1. Google Apps Scriptエディタの右上「**デプロイ**」ボタンをクリック
2. ドロップダウンメニューから「**新しいデプロイ**」を選択

### **4-2. デプロイタイプを選択**

1. 「種類の選択」の⚙️アイコンをクリック
2. 「**ウェブアプリ**」を選択

### **4-3. デプロイ設定を入力**

以下の設定を入力:

| 項目 | 設定値 |
|------|--------|
| **説明** | 介護記録作成アシスタント v1.0 |
| **次のユーザーとして実行** | 自分（あなたのメールアドレス） |
| **アクセスできるユーザー** | 自分のみ（または必要に応じて変更） |

### **4-4. デプロイを実行**

1. 「**デプロイ**」ボタンをクリック
2. 初回は承認画面が表示される

---

### **4-5. 承認プロセス（重要）**

#### **承認手順:**

1. 「**アクセスを承認**」ボタンをクリック
2. Googleアカウントを選択
3. 警告画面が表示される:
   ```
   このアプリは Google で確認されていません
   ```
4. 「**詳細**」をクリック（小さいリンク）
5. 「**（プロジェクト名）に移動**」をクリック
6. 権限の確認画面が表示される:
   - Google スプレッドシートのスプレッドシートの表示、編集、作成、削除
   - 外部サービスへの接続
7. 「**許可**」ボタンをクリック

#### **⚠️ なぜこの警告が出るのか:**

- Google Apps Scriptが個人開発のアプリであるため
- 自分で作成したスクリプトは安全です
- 公開する場合は別途Google審査が必要

---

### **4-6. デプロイ完了**

1. デプロイが完了すると、以下の情報が表示される:
   - **ウェブアプリ URL**: `https://script.google.com/macros/s/XXXXXXXX/exec`
   - **デプロイ ID**: `XXXXXXXX`

2. 「**ウェブアプリ URL**」の横の「**コピー**」ボタンをクリック

3. URLをブックマークに保存（後で使用）

#### **確認ポイント:**

✅ URLが `https://script.google.com/macros/s/` で始まっている  
✅ URLが `/exec` で終わっている  
✅ URLをコピー済み

---

## STEP 5: 初回設定とテスト

### **5-1. ウェブアプリを開く**

1. コピーしたウェブアプリURLをブラウザの新しいタブで開く
2. アプリの画面が表示される

### **5-2. API設定**

1. 黄色い「**⚙️ API設定**」カードが表示される
2. 「Google Gemini API Key」入力欄に、STEP 3でコピーしたAPIキーを貼り付け
3. 「**💾 保存してテスト**」ボタンをクリック
4. 数秒待つ

#### **成功の場合:**

```
✅ 接続成功！APIキーを保存しました
```

- API設定カードが閉じる
- 左上に「✅ 接続済み」と表示される

#### **失敗の場合:**

```
❌ 接続失敗: Invalid API Key
```

**対処法:**
- APIキーを再確認
- `AIza` で始まっているか確認
- コピー時に余分なスペースが入っていないか確認

---

### **5-3. テスト入力**

#### **テキスト入力でテスト:**

1. 「文字起こしテキスト」欄に以下を入力:
   ```
   田中様、今朝はばいたるそくていをじっししました。
   けつあつは130/80、たいおんは36.5度でした。
   しょくじかいじょをおこない、はいせつかいじょもじっししました。
   ```

2. 「📝 経過観察記録」ボタンをクリック

3. 「AI処理中... 30秒ほどお待ちください」と表示される

4. 約30秒後、生成された記録文が表示される:
   ```
   田中様、今朝はバイタル測定を実施しました。
   血圧は130/80、体温は36.5度でした。
   食事介助を行い、排泄介助も実施しました。
   ```

5. 「📋 クリップボードにコピー」ボタンをクリック

6. コピー完了のメッセージが表示される

#### **確認ポイント:**

✅ 用語が正しく変換されている（ばいたる → バイタル）  
✅ 記録文が生成されている  
✅ コピー機能が動作している

---

### **5-4. 音声入力テスト（デスクトップの場合）**

1. 「🎤 音声入力」カードのマイクボタンをクリック
2. ブラウザがマイク許可を求める → 「許可」をクリック
3. マイクボタンが赤くなる（録音中）
4. 何か話してみる（例: "こんにちは"）
5. リアルタイムでテキストが入力される
6. 再度マイクボタンをクリックして停止

#### **確認ポイント:**

✅ マイク許可が出た  
✅ 録音中にボタンが赤くなる  
✅ 話した内容が表示される

---

### **5-5. 保存機能テスト**

1. 「タイトル」欄に「テスト記録」と入力
2. 「💾 保存」ボタンをクリック
3. 「✅ 記録を保存しました」と表示される
4. 左サイドバーの「📚 記録履歴」に「テスト記録」が追加される

#### **確認ポイント:**

✅ 履歴に記録が表示される  
✅ クリックすると内容が読み込まれる  
✅ Google Sheetsに「介護記録履歴」シートが作成されている

---

### **5-6. Google Sheetsの確認**

1. Google Apps Scriptエディタに戻る
2. 左側のアイコンから「⚙️ プロジェクトの設定」をクリック
3. 「関連するスプレッドシート」を確認
4. リンクをクリックしてスプレッドシートを開く
5. 以下の2つのシートが作成されていることを確認:
   - **介護記録履歴** - 保存した記録が表示される
   - **用語辞書** - デフォルト70語が登録されている

## STEP 6: iPad/iPhone設定

### **6-1. URLの共有**

#### **方法A: メール経由**

1. PCでウェブアプリURLをコピー
2. 自分宛にメール送信
3. iPad/iPhoneでメールを開く
4. URLをタップ

#### **方法B: AirDrop（Apple製品のみ）**

1. PCのSafariでウェブアプリを開く
2. 共有ボタン → AirDrop
3. iPadを選択

#### **方法C: QRコード**

1. [QRコード生成サイト](https://www.qr-code-generator.com/)でURLからQRコード生成
2. iPad/iPhoneのカメラで読み取る

---

### **6-2. Safariで開く**

1. URLをSafariで開く
   - ⚠️ Chrome on iOSは音声認識非対応
2. アプリが表示される
3. API設定が既に完了している場合は、そのまま使用可能

---

### **6-3. ホーム画面に追加**

#### **手順:**

1. Safari画面下部の「**共有**」ボタン（□↑）をタップ
2. 「**ホーム画面に追加**」をタップ
3. 名前を編集（例: "介護記録"）
4. 「**追加**」をタップ
5. ホーム画面にアイコンが追加される

#### **メリット:**

- アプリのようにワンタップで起動
- フルスクリーンで表示
- ブラウザのアドレスバーが非表示

---

### **6-4. マイク許可設定**

#### **初回使用時:**

1. マイクボタンをタップ
2. 「このWebサイトがマイクへのアクセスを求めています」と表示
3. 「**許可**」をタップ

#### **許可しなかった場合:**

1. iPad/iPhoneの「**設定**」アプリを開く
2. 「**Safari**」をタップ
3. 「**カメラとマイク**」または「**マイク**」をタップ
4. 「**許可**」に変更
5. Safariを再起動

---

### **6-5. 音声入力テスト**

1. ホーム画面のアイコンをタップ
2. 「🎤 音声入力」のマイクボタンをタップ
3. 何か話してみる
4. リアルタイムでテキスト化される
5. 停止ボタンをタップ

#### **確認ポイント:**

✅ マイクボタンが赤くなる  
✅ 話した内容が表示される  
✅ 用語が自動変換される

## よくある質問

### **Q1: デプロイURLが開けない**

**A**: 以下を確認してください
1. URLが正しくコピーされているか
2. デプロイ時の承認を完了したか
3. 「アクセスできるユーザー」設定が適切か

**対処法:**
- デプロイを削除して再度デプロイ
- 別のブラウザで試す

---

### **Q2: API接続テストで失敗する**

**A**: 以下を確認してください
1. APIキーが `AIza` で始まっているか
2. コピー時に余分なスペースが入っていないか
3. Google Cloud Console で API が有効か

**対処法:**
1. [Google AI Studio](https://aistudio.google.com/app/apikey)で新しいキーを作成
2. 古いキーを削除
3. 新しいキーでテスト

---

### **Q3: 音声入力が動かない（iPad/iPhone）**

**A**: 以下を確認してください
1. **Safariを使用しているか**（Chrome非対応）
2. マイク許可を出したか
3. プライベートブラウズモードになっていないか
4. iOS 14.5以降か

**対処法:**
1. 設定 > Safari > マイク: 許可
2. プライベートブラウズを解除
3. iOSをアップデート

---

### **Q4: 履歴が表示されない**

**A**: 以下を確認してください
1. 記録を保存したか
2. Google Sheetsが作成されているか
3. ページをリロードしたか

**対処法:**
1. 「🔄 辞書を読み込む」ボタンをクリック
2. ページをリロード（F5 / Cmd+R）
3. Google Sheetsを確認

---

### **Q5: 用語辞書が効かない**

**A**: 以下を確認してください
1. 辞書を読み込んだか
2. 用語が登録されているか
3. 入力した音声が辞書の「誤認識語」と一致するか

**対処法:**
1. 「📖 用語辞書管理」で辞書を確認
2. 実際に誤認識された語を登録
3. 「🔄 辞書を読み込む」で再読み込み

---

### **Q6: デプロイURLを忘れた**

**A**: 以下の手順で再確認できます
1. Google Apps Scriptエディタを開く
2. 「デプロイ」→「デプロイを管理」
3. 最新のデプロイのURLをコピー

---

### **Q7: 複数デバイスで使いたい**

**A**: 
- **同じGoogleアカウント**でログインすれば、どのデバイスでも同じデータにアクセス可能
- URLをブックマーク/ホーム画面に追加
- API設定は一度だけでOK

---

### **Q8: 同僚と共有したい**

**A**: デプロイ設定を変更
1. 「デプロイ」→「デプロイを管理」
2. 編集アイコン（✏️）をクリック
3. 「アクセスできるユーザー」を変更:
   - 全員（匿名ユーザーを含む）
   - ドメイン内のユーザー
   - 特定のユーザー
4. 「デプロイ」をクリック
5. 新しいURLを共有

⚠️ **注意**: データは作成者のGoogle Driveに保存されます

---

### **Q9: データをバックアップしたい**

**A**: Google Sheetsをダウンロード
1. Google Sheetsを開く
2. 「ファイル」→「ダウンロード」
3. 形式を選択:
   - Excel (.xlsx)
   - CSV (.csv)
   - PDF

---

### **Q10: エラーが出続ける**

**A**: 最終手段
1. プロジェクト全体を削除
2. 最初からやり直す
3. 新しいGoogleアカウントで試す
